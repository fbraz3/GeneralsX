## 2026-05-11: Fix FPS uncapped and frame counter instability (Issue #132)

Fixed two critical timing bugs affecting both Generals and Zero Hour that caused FPS to be uncapped or frame counter to appear erratic on non-Windows platforms.

What was done:
- identified `FramePacer.cpp` using Windows-only `timeBeginPeriod()` / `timeEndPeriod()` API without `#ifdef _WIN32` guards, causing undefined behavior on macOS/Linux.
- added `#ifdef _WIN32` guards around timer precision calls in `FramePacer::FramePacer()` and `~FramePacer()`.
- identified `FrameRateLimit::wait()` missing validation for `maxFps == 0` or `maxFps > 1000000` (uncapped/invalid mode), causing division-by-zero underflow.
- added early-exit uncapped-mode handling that skips frame limiting when FPS target is 0 or unreasonably high (1M+).
- ensured shared `Core/GameEngine/` fixes automatically apply to both Generals base and Zero Hour expansions.

Validation:
- compiled both GeneralsX and GeneralsXZH successfully without errors.
- manually tested GeneralsXZH launch and confirmed stable frame pacing (no uncapped spike).
- manually tested GeneralsX launch and confirmed stable frame pacing (no uncapped spike).

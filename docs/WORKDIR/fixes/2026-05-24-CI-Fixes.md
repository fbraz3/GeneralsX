## Issues Identified

### 1. macOS: Invalid Cast of THREAD_ID
**Files:** `Core/Libraries/Source/WWVegas/WWDebug/wwmemlog.cpp`, `wwprofile.cpp`

**Problem:** On macOS, `GetCurrentThreadId()` returns `_opaque_pthread_t*` (pointer), but code was casting to `int`.

**Fix Applied:**
- Changed `ThreadID` type from `unsigned long` to `void*` in `wwprofile.cpp`
- Removed `static_cast<int>()` in `wwmemlog.cpp`

### 2. Linux: Invalid Cast of HKL

**Status:** Already fixed with comment `// GeneralsX @bugfix GitHub Copilot 20/05/2026 Use pointer-sized cast for HKL on x64 MinGW.`

## Prevention Strategy

### Global Pattern: Platform-Specific Type Handling

**Problem:** Code assumes `GetCurrentThreadId()` returns `DWORD` (Windows), but on macOS it returns a pointer.

**Solution:** Use `void*` for cross-platform thread IDs when comparing with `GetCurrentThreadId()`.

### Pattern for HKL/Keyboard Layout

**Problem:** `GetKeyboardLayout()` returns pointer-sized value on x64 MinGW.

**Solution:** Always use `reinterpret_cast<ULONG_PTR>()` before casting to integer types.

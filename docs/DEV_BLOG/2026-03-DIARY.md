## 2026-03-03 (SESSION 70): Fix macOS Build Failure — Vulkan SDK Download (UPDATED AGAIN)

**Second failure**: GitHub Actions revealed actual SDK structure.

**New Discovery**:
- The ZIP archive structure is NOT a `.app` bundle as initially assumed
- Actual structure: `vulkansdk-macOS-1.4.341.1.zip` contains a single executable file: `vulkansdk-macOS-1.4.341.1`
- This is a direct command-line installer (not wrapped in `.app` bundle)
- File is already executable (mode 755)

**Updated Fixes**:

1. **`.github/workflows/build-macos.yml`** — Fixed installer detection strategy:
   - **Strategy 1**: Look for executable directly: `vulkansdk-macOS-*` (type: file, executable)
   - **Strategy 1b**: If found but not executable, make it executable with `chmod +x`
   - **Strategy 2**: Fallback to `.app` bundle if direct executable not found (`*.app/Contents/MacOS/InstallVulkan`)
   - **Diagnostics**: List extracted contents and full file tree if nothing found
   - All strategies now properly capture exit status from installer

2. **Corrected installer invocation**:
   ```bash
   ./vulkansdk-macOS-1.4.341.1 \
     --root "$HOME/VulkanSDK/1.4.341.1" \
     --accept-licenses \
     --default-answer \
     --confirm-command install
   ```
   - Same parameters, different executable type (direct file vs app bundle)

3. **Improved error handling**:
   - Shows extracted file listing if installer not found
   - Prints up to 20 files in extracted structure for debugging
   - Captures installer exit status correctly (was broken before)

**Lessons**:
- macOS `.app` bundles are not always used for installers; direct executables are simpler
- Zip structure discovery must handle both patterns: bundled vs direct executables
- Exit status capture placement matters: must be immediately after command invocation

**Objective**: Fix GitHub Actions `build-macos` workflow failure: `hdiutil: attach failed - image not recognized`.

**Root Cause Analysis**:
- Original URL `https://sdk.lunarg.com/download/latest/mac/vulkan-sdk.dmg` was returning redirects or HTTP errors (~1426 bytes HTML).
- LunarG's SDK delivery changed: latest version (1.4.341.1) is now distributed as `.zip` containing a macOS app installer bundle.
- Correct direct URL: `https://sdk.lunarg.com/sdk/download/1.4.341.1/mac/vulkansdk-macos-1.4.341.1.zip` (318 MB).
- SHA256 hash for validation: `632cbe96c8ed6ed00c6ce25e3a7738c466134f76586e1c51f1419410d7f9042e`.

**Architecture of Vulkan SDK 1.4.341.1**:
- ZIP contains: `vulkansdk-macOS-1.4.341.1.app` (macOS application bundle)
- Inside `.app`: `/Contents/MacOS/InstallVulkan` (executable installer script)
- Installer accepts command-line parameters for unattended deployment
- Target install path: `$HOME/VulkanSDK/1.4.341.1/` (version-specific directory)

**Fixes Applied**:

1. **`.github/workflows/build-macos.yml`** — Updated Vulkan SDK installation strategy (REVERSED PRIORITY):
   - **Strategy 1 (PREFERRED)**: Direct download of versioned SDK (1.4.341.1) with SHA256 validation
     - Download from: `https://sdk.lunarg.com/sdk/download/1.4.341.1/mac/vulkansdk-macos-1.4.341.1.zip`
     - Validate SHA256: `632cbe96c8ed6ed00c6ce25e3a7738c466134f76586e1c51f1419410d7f9042e`
     - Extract ZIP to find `.app` bundle
     - Execute installer: `./vulkansdk-macOS-1.4.341.1.app/Contents/MacOS/InstallVulkan`
     - Install to: `$HOME/VulkanSDK/1.4.341.1/`
     - Retry up to 3 times with 15s delays
   - **Strategy 2 (FALLBACK)**: Homebrew (`brew install --cask vulkan-sdk`)
     - Only attempted if versioned download fails
     - May install older version than 1.4.341.1, but guarantees working installation
     - Better than nothing if LunarG CDN is unreachable
   - **Why reversed**: Versioned download is more reliable (direct URL, SHA256 validated, specific version known to work); Homebrew is fallback in case LunarG CDN is down

2. **Installation parameters**:
   - `--root $HOME/VulkanSDK/1.4.341.1` — version-specific path (supports coexistence)
   - `--accept-licenses` — non-interactive (skip dialog)
   - `--default-answer` — use presets for all prompts
   - `--confirm-command install` — execute install (not just preview)

3. **`cmake/dx8.cmake`** — Updated Vulkan SDK detection paths:
   - First: Check `$VULKAN_SDK` environment variable
   - Second: Search `$HOME/VulkanSDK/*/` directories (find latest version)
   - Third: Try Homebrew locations (Intel: `/usr/local/Caskroom/vulkan-sdk`, ARM64: `/opt/homebrew/Caskroom/vulkan-sdk`)
   - Validate MoltenVK: `lib/libMoltenVK.dylib` must exist
   - Pass detected path to Meson via `VULKAN_SDK` env var

4. **Diagnostics & Error Handling**:
   - SHA256 mismatch detected early (prevent .app corruption)
   - File size checks reveal HTML error responses (< 10MB)
   - First 20 lines of invalid files printed for debugging
   - Installer failures reported with exit status
   - Graceful fallback if primary strategy fails
   - Warns if MoltenVK not found (system Vulkan may still work)

**Testing in CI**:
1. GitHub Actions will retry 3 times on download failure (15s between attempts)
2. SHA256 validation ensures downloaded file integrity
3. Logs show which strategy succeeded (Homebrew vs manual installer)
4. MoltenVK verification confirms complete installation

**Lessons Learned**:
- LunarG changed distribution format (DMG→ZIP with app installer) — needed URL and extraction method updates.
- macOS app installers need explicit `--accept-licenses --confirm-command` for headless operation.
- Version-specific paths ($HOME/VulkanSDK/1.4.341.1/) better than generic ($HOME/VulkanSDK/) for multi-version coexistence.
- SHA256 validation critical for supply-chain security (detect corrupted/HTML responses immediately).
